1. Présentation du Projet

TimeTrack BGFIBank est une application web de gestion et de pointage des heures de travail développée pour BGFIBank Gabon. Elle permet à chaque agent de déclarer ses sessions de travail en les associant aux tâches, processus et objectifs stratégiques de la banque. Les chefs de département valident les sessions, et l'administrateur dispose d'un tableau de bord de productivité global avec export des données.

Objectifs Fonctionnels

  • Permettre aux agents de pointer leurs heures par tâche (chronomètre ou saisie manuelle)
  • Offrir aux chefs de département un outil de validation/rejet des sessions
  • Donner à l'administrateur une vue complète de la productivité (objectifs, départements, agents)
  • Gérer le référentiel organisationnel : départements, objectifs stratégiques, processus, tâches
  • Tracer toutes les actions via un journal d'audit sécurisé
  • Protéger les accès avec un système d'authentification JWT et anti-force brute
  • Permettre la réinitialisation sécurisée des mots de passe via code temporaire

8. Système de Suppression avec Confirmation NOUVEAU v3.0

Problème résolu

Avant : Les boutons de suppression utilisaient confirm() natif du navigateur, bloqué dans certains environnements (Cloudflare Workers, iframes). Résultat : les boutons ne répondaient pas. De plus, les routes DELETE manquaient côté backend, retournant des erreurs 404. Enfin, les endpoints GET retournaient tous les éléments y compris les inactifs, donc les suppressions n'avaient aucun effet visible.

Solution implémentée

Après : (1) Modal HTML/CSS personnalisé showConfirmDialog(), (2) 5 routes DELETE ajoutées au backend, (3) Filtrage WHERE status='Actif' sur les GET.

Fonctionnement du Modal de Confirmation

Affichage du modal

Quand l'utilisateur clique sur l'icône 🗑 :

  1. Un overlay semi-transparent est inséré dans le DOM
  2. La carte modale affiche une icône rouge, un titre « Confirmer la suppression », le nom de l'élément
  3. Deux boutons : Annuler (gris) et Oui, supprimer (rouge)
  4. La touche Échap ferme le modal (équivalent Annuler)
// Fonction générique de confirmation (admin.js)
function showConfirmDialog(message, onConfirm) {
  const overlay = document.createElement('div')
  overlay.style.cssText = 'position:fixed;inset:0;background:rgba(0,0,0,0.5);...'
  overlay.innerHTML = `
    <div class="confirm-modal">
      <div class="icon">🗑</div>
      <h3>Confirmer la suppression</h3>
      <p>${message}</p>
      <button id="btn-cancel">Annuler</button>
      <button id="btn-confirm">Oui, supprimer</button>
    </div>`
  
  document.getElementById('btn-confirm').onclick = () => {
    overlay.remove(); onConfirm()  // Appel de la suppression
  }
  document.getElementById('btn-cancel').onclick = () => overlay.remove()
  document.addEventListener('keydown', e => { if(e.key==='Escape') overlay.remove() })
}

9. Code Reset — Réinitialisation Sécurisée de Mot de Passe NOUVEAU v3.0

Pourquoi ce mécanisme ?

Problème sans code reset : L'admin devait saisir un nouveau mot de passe en clair, le communiquer oralement à l'agent (risque d'écoute), et l'agent ne changeait souvent pas ce mot de passe temporaire. L'admin connaissait donc en permanence les mots de passe de tous les agents.
Avec le code reset : L'admin génère un code à 6 chiffres valable 30 minutes. L'agent utilise ce code pour définir lui-même son nouveau mot de passe. L'admin ne voit jamais le nouveau mot de passe. Le code est détruit après utilisation.

Flux Complet — Code Reset

Étape 1 — Côté Administrateur

  1. Aller dans Gestion des Utilisateurs
  2. Cliquer sur l'icône 🔑 (clé) à côté de l'agent concerné
  3. Un modal affiche le code généré, ex : 482917
  4. L'admin communique ce code à l'agent (en personne, par téléphone)

Étape 2 — Côté Agent

  1. Aller sur la page de connexion /login
  2. Cliquer sur « J'ai un code de réinitialisation »
  3. Saisir son email, le code reçu, et un nouveau mot de passe
  4. Cliquer sur Confirmer
  5. Le système valide le code, change le mot de passe, débloque l'IP si nécessaire
  6. Redirection automatique vers la page de connexion après 3 secondes

Caractéristiques Techniques du Code Reset

ParamètreValeurRaison
Format du code6 chiffres numériquesSimple à communiquer verbalement
Durée de validité30 minutesSuffisant sans être trop long
UsageÀ usage uniqueDétruit après utilisation, ne peut pas être réutilisé
StockageMap en mémoire (resetCodes)Aucune trace en base (sécurité)
Effet sur IP bloquéeDébloque automatiquementL'agent ne doit pas attendre 2 min après un reset
AuditLoggé (RESET_PASSWORD_REQUEST / RESET_PASSWORD_CONFIRM)Traçabilité complète

Interaction avec le Rate Limiting

Si un agent oublie son mot de passe après 3 tentatives échouées, son IP est bloquée 2 minutes. Pendant ce blocage, le panneau reset s'ouvre automatiquement sur la page de connexion avec le message : « Votre IP est bloquée. Vous pouvez utiliser un code de réinitialisation. »

Quand l'agent entre un code reset valide : le blocage IP est levé immédiatement (appel à resetAttempts(ip)), son mot de passe est changé, et il peut se reconnecter sans attendre. 

// Backend : reset-confirm lève le blocage IP
app.post('/api/auth/reset-confirm', async (c) => {
  const ip = c.req.header('CF-Connecting-IP') || 'unknown'
  // ... validation email + code ...
  
  // Mettre à jour le mot de passe
  await DB.prepare('UPDATE users SET password_hash=?, password_encrypted=? WHERE id=?')
    .bind(passwordHash, passwordEncrypted, user.id).run()

  resetCodes.delete(user.id)   // Code détruit (usage unique)
  resetAttempts(ip)            // ← DÉBLOCAGE IP AUTOMATIQUE

  return c.json({ message: 'Mot de passe modifié avec succès' })
})
