Section 01

Présentation Générale du Projet

1.1 Contexte et objectifs

TimeTrack BGFIBank est une application web interne développée pour la BGFIBank afin de permettre le suivi en temps réel du temps de travail des collaborateurs. Elle répond au besoin de mesurer la contribution effective de chaque agent aux objectifs stratégiques de la banque.

L'application permet à chaque employé de saisir, démarrer et arrêter des sessions de travail associées à des tâches définies, des processus métiers et des objectifs stratégiques. Les données collectées alimentent des tableaux de bord analytiques pour l'encadrement et la direction.

Objectif principal : Mesurer et optimiser la productivité des agents en calculant le ratio entre les heures productives saisies et la capacité journalière standard de 8 heures, par agent, département et objectif stratégique.

4.2 Jointures principales par fonctionnalité

🔐 Authentification — Récupération d'un utilisateur avec son département LEFT JOIN
SELECT u.*, d.name as department_name FROM users u LEFT JOIN departments d ON u.department_id = d.id WHERE u.email = ? AND u.status = 'Actif'

Le LEFT JOIN permet de récupérer les utilisateurs sans département (ex: Administrateur) en retournant NULL pour department_name.

📊 Rapport Admin — Sessions avec toutes les informations jointes JOIN multiple
SELECT ws.*, u.first_name || ' ' || u.last_name as agent_name, d.name as department_name, t.name as task_name, p.name as process_name, o.name as objective_name, o.color as objective_color FROM work_sessions ws JOIN users u ON ws.user_id = u.id JOIN departments d ON ws.department_id = d.id JOIN tasks t ON ws.task_id = t.id JOIN processes p ON t.process_id = p.id JOIN strategic_objectives o ON ws.objective_id = o.id ORDER BY ws.start_time DESC

Cette requête relie 6 tables en chaîne : sessions → agents → départements → tâches → processus → objectifs. Utilisée pour les exports de rapports complets.

🎯 Stats Admin — Heures cumulées par objectif stratégique LEFT JOIN + GROUP BY
SELECT o.name, o.color, o.target_percentage, COALESCE(SUM(ws.duration_minutes), 0) as total_minutes FROM strategic_objectives o LEFT JOIN work_sessions ws ON ws.objective_id = o.id AND ws.status = 'Validé' -- ne compter que les sessions validées WHERE o.status = 'Actif' GROUP BY o.id, o.name, o.color, o.target_percentage ORDER BY total_minutes DESC

Le LEFT JOIN garantit que tous les objectifs apparaissent même s'ils n'ont aucune session enregistrée (total_minutes = 0 via COALESCE).

📈 Productivité Chef — Heures productives/non-productives par agent aujourd'hui LEFT JOIN + CASE WHEN
SELECT u.id, u.first_name || ' ' || u.last_name as agent_name, COALESCE(SUM(ws.duration_minutes), 0) as productive_minutes_today, CASE WHEN 480 - COALESCE(SUM(ws.duration_minutes), 0) > 0 THEN 480 - COALESCE(SUM(ws.duration_minutes), 0) ELSE 0 END as non_productive_minutes_today FROM users u LEFT JOIN work_sessions ws ON ws.user_id = u.id AND ws.status = 'Validé' AND date(ws.start_time) = date('now') -- filtre sur le jour courant WHERE u.department_id = ? AND u.role = 'Agent' AND u.status = 'Actif' GROUP BY u.id, u.first_name, u.last_name

Logique métier : 480 minutes = 8h journée standard. CASE WHEN remplace GREATEST() absent de SQLite. Seules les sessions en statut Validé comptent comme productives.

📋 Tâches disponibles pour un agent — Jointure 4 niveaux JOIN chaîné
SELECT t.*, p.name as process_name, o.name as objective_name, o.color as objective_color FROM tasks t JOIN processes p ON t.process_id = p.id JOIN strategic_objectives o ON t.objective_id = o.id WHERE t.department_id = ? -- département de l'agent connecté AND t.status = 'Actif' ORDER BY o.name, t.name

⚠️

Note pour une évolution future

Le SHA-256 sans sel est fonctionnel pour un environnement interne protégé. Pour un déploiement internet public, il est recommandé de migrer vers bcrypt ou Argon2 avec sel aléatoire par utilisateur.

6.3 Rate limiting & verrouillage de compte

🛡️

Protection contre les attaques par force brute

Après 3 tentatives de connexion échouées depuis la même adresse IP, le compte est verrouillé pendant 15 minutes. Les tentatives restantes sont affichées à l'utilisateur. Le verrou se lève automatiquement après expiration.

Section 07

Calcul de Productivité

7.1 Formule de base

Règle métier : Une journée de travail standard est de 8 heures = 480 minutes.
Heures productives = Somme des minutes des sessions validées du jour (plafonnée à 480 min)
Heures non-productives = MAX(0, 480 − Heures productives)

7.2 Tableau de calcul — Exemples

Sessions validéesDurée totaleH. productivesH. non-productivesTaux productivité
Aucune session0 min0h 00m8h 00m0 %
2 sessions120 min2h 00m6h 00m25 %
3 sessions360 min6h 00m2h 00m75 %
4 sessions480 min8h 00m0h 00m100 %
Heures sup.540 min8h 00m (plafonné)0h 00m100 %

7.3 Seuils de performance et badges

Taux de productivitéBadge affichéCouleurSignification
≥ 80 %✅ BonVertJournée productive
50 % – 79 %⚠️ MoyenOrangeJournée partiellement productive
< 50 %❌ FaibleRougeJournée insuffisante

7.4 Où est affiché ce calcul ?

  • Tableau de bord Admin : Bloc "Productivité" global + tableau détaillé par agent avec barre de progression
  • Tableau de bord Chef de Département : Bloc productivité de l'équipe + tableau par agent avec heures productives/non-productives du jour
  • Tendance mensuelle Admin : Graphique courbe sur 6 mois des heures validées

7.5 Précision technique — CASE WHEN au lieu de GREATEST()

-- SQLite ne supporte pas GREATEST(), on utilise CASE WHEN : CASE WHEN 480 - COALESCE(SUM(duration_minutes), 0) > 0 THEN 480 - COALESCE(SUM(duration_minutes), 0) ELSE 0 END AS non_productive_minutes
